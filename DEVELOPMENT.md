# WakeyWakey Development Guide

Build and contribute to WakeyWakey.

## Clone the Repository

```bash
git clone git@github.com:brndnsvr/WakeyWakey.git
cd WakeyWakey
```

## Requirements

- macOS 15.0 or later
- Xcode with macOS 15+ SDK
- Homebrew
- XcodeGen (`brew install xcodegen`)
- create-dmg (`brew install create-dmg`) — for releases only

## Initial Setup

```bash
brew install xcodegen
```

Set your Apple Developer Team ID (required for code signing):

```bash
export WAKEY_TEAM_ID="YOUR_TEAM_ID"
```

Add to `~/.zshrc` to persist. Then generate the Xcode project:

```bash
./scripts/generate_project.sh
```

## Build & Run

```bash
# Debug (faster iteration)
./scripts/build_debug.sh
./scripts/install.sh
./scripts/run.sh

# Release (Developer ID signed — use for /Applications installs)
./scripts/build.sh
./scripts/install.sh
./scripts/run.sh
```

Both build scripts compile two targets: the `WakeyWakey` app and the `wakey` CLI tool.

First launch will prompt for Accessibility permission. If the dialog doesn't appear:
1. System Settings → Privacy & Security → Accessibility
2. Add WakeyWakey and enable it
3. Relaunch the app

To stop the app:

```bash
./scripts/kill.sh
```

## Release Process

### Create DMG only

```bash
./scripts/release.sh 1.2.0
```

### Create DMG + publish to GitHub

```bash
./scripts/release.sh 1.2.0 --publish
```

This will:
1. Build both targets (Release config)
2. Embed the `wakey` CLI in the app bundle
3. Re-sign the app bundle
4. Create the DMG with drag-to-Applications
5. Notarize and staple the DMG
6. Tag the release, push tag, create GitHub release with DMG attached

To skip the build step (use existing build artifacts):

```bash
SKIP_BUILD=true ./scripts/release.sh 1.2.0 --publish
```

### Update Homebrew Tap

After publishing a GitHub release:

1. Get the SHA256:

```bash
shasum -a 256 dist/WakeyWakey-X.Y.Z.dmg
```

2. Edit the cask formula (installed tap location):

```bash
$EDITOR /opt/homebrew/Library/Taps/brndnsvr/homebrew-tap/Casks/wakeywakey.rb
```

Update `version` and `sha256`.

3. Commit and push:

```bash
cd /opt/homebrew/Library/Taps/brndnsvr/homebrew-tap
git add Casks/wakeywakey.rb
git commit -m "update wakeywakey to X.Y.Z"
git push
```

## Architecture

### Targets

| Target | Type | Output |
|--------|------|--------|
| WakeyWakey | application | `.app` bundle (menu bar app) |
| wakey | tool | CLI binary (standalone Mach-O) |

### IPC

The CLI communicates with the running app via **CFMessagePort** (local Mach ports). The app creates a port named `com.brndnsvr.WakeyWakey.cli` on launch. The CLI connects, sends a JSON command, and receives a synchronous JSON response.

### Jiggle Animation

Animated multi-waypoint movement (not instant teleport):
- **Path types**: Arc (40%), zigzag (35%), direct (25%)
- **Waypoints**: 4-8 steps per jiggle
- **Total distance**: 15-35 pixels
- **Duration**: 0.5-1.0 seconds
- **Center bias**: 52% of jiggles move toward screen center
- Constrained to current screen bounds

### Key APIs

- **IOKit IOPM assertions** — prevent system idle sleep and display sleep
- **ApplicationServices CGEvents** — post mouse movements
- **ServiceManagement SMAppService** — Launch at Login
- **AXIsProcessTrustedWithOptions** — auto-open Accessibility settings
- **CFMessagePort** — IPC between CLI and app
- **Combine** — settings observation and menu title updates

### Idle Detection

Takes minimum across explicit HID event types for robustness with DisplayLink/virtual displays. Also tracks mouse position changes between ticks to detect Universal Control cursor movement.

### Settings

`Settings.swift` is a singleton backed by UserDefaults with `@Published` properties. Configurable values: timer durations (3 presets), idle threshold, jiggle interval min/max. All have sensible defaults and a `resetToDefaults()` method.

## Troubleshooting (Development)

- **No menu bar icon** — Ensure `LSUIElement=true` in Info.plist and running from /Applications
- **Menu not responding** — Check `button.target` and `button.action` are set
- **Jiggle during typing** — Idle detection uses min across all event types
- **Cursor jumps to primary display** — Coordinates must be clamped to current screen
- **CLI can't connect** — App must be running; check `CLIServer` initialized in `applicationDidFinishLaunching`

## Technical Notes

- Uses AppKit (not SwiftUI) with a required `main.swift` for proper menu bar init
- `NSApp.setActivationPolicy(.accessory)` must be in `init()`, NOT `applicationDidFinishLaunching`
- The deprecated `popUpMenu` method is used intentionally — it works reliably
- 52% center bias prevents cursor drift to screen edges over long sessions
- Debug builds use a different signing identity which causes Accessibility permission re-prompts on upgrade — use Release builds for /Applications
